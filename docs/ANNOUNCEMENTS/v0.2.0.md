# lamboot-tools 0.2.0

Today we're releasing **lamboot-tools 0.2.0**, the first public version of our
boot-management toolkit for Linux. It's the toolkit we've been building
alongside the LamBoot bootloader — and the one we've been using ourselves to
diagnose, migrate, and recover boot configurations on real machines.

## What it is

`lamboot-tools` is eleven tools sharing one philosophy: **diagnose before you
repair, show before you execute, confirm before you destroy.** Every mutating
operation is preceded by a plan you can read, and every scan emits structured
JSON you can pipe into your own tooling.

The headline tool is **`lamboot-migrate`**, the first tool we know of that
automates Linux BIOS→UEFI conversion end-to-end. Ten phases. Seven preflight
guardrails (hybrid MBR, Windows dual-boot, LVM/dm-crypt root, insufficient ESP
space, dangerous fstab entries, missing tools, already-UEFI). Distro-aware
bootloader install for apt, dnf, pacman, zypper, and a generic fallback. A
`verify` subcommand with eleven post-migration checks. A `rollback` subcommand
if you ever need to undo.

Beyond migration, there's:

- **`lamboot-diagnose`** — 11-category UEFI boot-chain scanner. ~30 checks.
  Detects LamBoot, GRUB, systemd-boot, rEFInd, Limine, and Windows Boot Manager
  with PE32+ validation. Works offline against a disk image.
- **`lamboot-esp`** — ESP health, inventory, and cleanup with
  bootloader-critical safety (never removes active bootloader binaries, UKIs,
  BLS entries, or fallback path files).
- **`lamboot-backup`** — NVRAM snapshots with `save`/`restore`/`show`/`list`.
- **`lamboot-repair`** — six-phase guided repair (diagnose → plan → show →
  confirm → execute → verify) with risk tiers you can cap.
- **`lamboot-doctor`** — single-sudo-escalation guided orchestrator
  (beta).
- **`lamboot-uki-build`** — host-side Unified Kernel Image builder wrapping
  `ukify`/`objcopy`/`sbsign` (beta).
- **`lamboot-signing-keys`** — Secure Boot key lifecycle with RSA-2048
  enforcement on MOK-enrolled keys per Debian bug #1013320 (experimental).
- **`lamboot-toolkit`** — the dispatcher that ties them all together.

There's also a **Proxmox companion** (`lamboot-toolkit-pve`) with per-VM setup,
fleet orchestration, NVRAM health monitoring, and OVMF VARS building.

## Design principles

- **Zero external dependencies beyond coreutils + util-linux + efibootmgr.** No
  Python runtime needed for the core tools. Two Python helpers
  (`lamboot-inspect`, `lamboot-pve-monitor`) are mirrored in from the
  LamBoot repo at release-build time.
- **Unified JSON schema v1** with dotted-path finding IDs, severity/status
  orthogonality, remediation URLs for every warning+ finding. Finding IDs are
  **semver-stable within the major version** — no silent renames.
- **Documented exit codes.** `0=ok`, `1=error`, `2=partial`, `3=noop`,
  `4=unsafe`, `5=abort`, `6=not-applicable`, `7=prerequisite`. See
  `man lamboot-tools` for the whole table.
- **Offline mode** for diagnose / esp / repair (v0.2) — operate against a disk
  image, not the running system.
- **Six-phase mutating pattern** for every tool that writes to NVRAM or the
  ESP. We show the plan before we execute it.

## Install

Fedora / RHEL / CentOS Stream / Rocky / Alma via Copr:

    sudo dnf copr enable lamco/lamboot-tools
    sudo dnf install lamboot-tools

Proxmox companion (on PVE nodes):

    sudo dnf copr enable lamco/lamboot-toolkit-pve
    sudo dnf install lamboot-toolkit-pve

Standalone `lamboot-migrate` (one-shot Linux BIOS→UEFI migration, no toolkit
install):

    sudo dnf copr enable lamco/lamboot-migrate
    sudo dnf install lamboot-migrate

Source tarball + signed checksums at
[github.com/lamco-admin/lamboot-tools/releases/v0.2.0](https://github.com/lamco-admin/lamboot-tools/releases/v0.2.0).

## First run

    $ sudo lamboot-diagnose

Or the guided version:

    $ sudo lamboot-doctor

For BIOS→UEFI migration on a test VM:

    $ sudo lamboot-migrate to-uefi --plan        # shows what it would do
    $ sudo lamboot-migrate to-uefi               # runs with confirmation
    $ sudo lamboot-migrate verify                # 11-check post-migration

Always `lamboot-backup save` before running anything destructive. The toolkit
asks before it writes, but belt-and-braces.

## What's next

We've committed to three future milestones (`ROADMAP.md`):

- **v0.3**: doctor + uki-build + pve-setup graduate to stable; `--offline
  DISK` for `lamboot-migrate`; Ubuntu/Debian PPA published.
- **v0.5**: signing-keys + pve-fleet graduate to stable; Homebrew tap for
  Linux-from-macOS admins.
- **v1.0**: Debian upstream submission for `lamboot-migrate`; Proxmox Phase 3
  native config option submission or companion upgrade; `lamboot-migrate`
  formal spin-off with its own product page and distro package.

We also have a public fleet-test matrix (`docs/FLEET-TEST-PLAN.md`) — 26 VMs
across five distros running every release against fresh fixtures. Tier 1 gates
every release. We publish the results nightly.
